SnazzyTableViewDataSource

public class SnazzyTableViewDataSource: NSObject, UITableViewDataSource

An instance of SnazzyTableViewDataSource is responsable for responding the methods that the collection view needs in order to display all the cells properly. You must have this instance in your desired UIViewController.

  • configFiles

    All the configFiles are going to be stored in this property. Most of the time you would use the convinience methods to insert and delete cells from the UITableView but in some cases you may need to access some of the configFiles stored in this property, that’s why it’s pubic.

  • insertRows(configFiles:with:)

    This method will insert all the cells specified at configFiles. The configFiles don’t need to be sorted by sections but the order of the array is gonna be the order of the cells for every section.

    For example if you insert these configFiles for sections:

    [ConfigFile 1: Section 1] [ConfigFile 2: Section 2] [ConfigFile 3: Section 2] [ConfigFile 4: Section 1] [ConfigFile 5: Section 3] [ConfigFile 6: Section 1]

    Then the order of the cells are going to be:

    ConfigFile 1 ConfigFile 4 ConfigFile 6 ConfigFile 2 ConfigFile 3 ConfigFile 5

  • getIndexPath(by:)

    This method will retrieve the IndexPath given by the filter. If the filter don’t match any result it will return nil, on the other hand if the filter matches multiple results it will retrieve the first one. You can use the indexPath after that to reload that cell or another operations that require the indexPath.

    Declaration

    Swift

    public func getIndexPath(by filter: (SnazzyTableCellConfigurator)->Bool) -> IndexPath?

    by

    A Closure that represents the cell that you want to find its IndexPath.

    Return Value

    The IndexPath that matched the given filter.
